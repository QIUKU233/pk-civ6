    - TEXT 这个可填的一般都是各种变量名，包括文本变量名

## 单位升级相关表

    - 常规晋级树是自己再到UnitPromotionPrereqs给各个升级能力进行排序，这样常规的技能树就ok了，
    - 另一种是随机晋级树不在UnitPromotionPrereqs排序，直接保持技能树同级，然后在到单位Units表的参数NumRandomChoices来选择自己一次升级随机几个技能，例如NumRandomChoices=3，就表示从晋级树里随机抽3个给你选择一个晋级
    - 注意NumRandomChoices等于0就是随机晋级树模式，大于就是随机晋级（应该不能为负数，感兴趣自己试一试,手动滑稽）

## 建筑限制相关表
### Building_RequiredFeatures
- 建筑在建造前置的地貌特征，或者说建筑的地貌前置表

- 貌似主要用于奇观建筑，普通建筑也可用。一般建筑的位置都是通过绑区域来确认位置

- 注意：这个表一个建筑分开填多个地貌，可限定多个地貌
- 貌似主要用于奇观建筑，普通建筑也可用。一般建筑的位置都是通过绑区域来确认位置
- Building_ValidFeatures只写一个类型会不会和Building_RequiredFeatures写一个效果一样呢

### Boosts
- 猜测尤里卡？？初步研究猜测：
    - Boosts数据表记录了在文明6中可以用来触发科技、文化和其他进步所需的特定条件。
    - 每个触发条件都有一个相应的ID和描述，并且与它相关联的科技、文化、单位、建筑、资源等都被包含在数据表中。
    - 当条件满足时，它将会给出一个触发增益的指令或者触发其他操作。

| 列名 | 数据类型 | 描述 |
| ---- | -------- | ---- |
| BoostID | INTEGER | 增益ID |
| TechnologyType | TEXT | 科技类别名称 |
| CivicType | TEXT | 政治制度类别名称 |
| Boost | INTEGER | 触发增益值 |
| TriggerId | INTEGER | 触发条件ID |
| TriggerDescription | TEXT | 触发条件描述 |
| TriggerLongDescription | TEXT | 触发条件详细描述 |
| Unit1Type | TEXT | 作用单位1类别名称 |
| BoostClass | TEXT | 触发增益类别名称 |
| Unit2Type | TEXT | 作用单位2类别名称 |
| BuildingType | TEXT | 建筑物类别名称 |
| ImprovementType | TEXT | 改良类别名称 |
| BoostingTechType | TEXT | 提升科技类别名称 |
| ResourceType | TEXT | 资源类别名称 |
| NumItems | INTEGER | 数量 |
| DistrictType | TEXT | 地区类别名称 |
| RequiresResource | BOOLEAN | 是否需要资源 |
| RequirementSetId | TEXT | 需求集ID |
| GovernmentSlotType | TEXT | 政府插槽类型名称 |
| BoostingCivicType | TEXT | 提升政治制度类别名称 |
| GovernmentTierType | TEXT | 政府阶层类型 |

### BoostHandlers
- 初步研究猜测：
    - BoostHandlers数据表包括了所有针对Boosts数据表中特定触发条件的处理程序。
    - 每个处理程序都有一个唯一的ID，并且它可以与一个Lua脚本、一个行为树或其他逻辑元素相关联。
    - 当触发条件被满足时，相应的处理程序将会被激活，执行相应的处理逻辑。

| 列名 | 数据类型 | 描述 |
| ---- | -------- | ---- |
| HandlerId | TEXT | 处理程序ID |
| TechBoostType | TEXT | 科技触发增益类别名称 |
| BehaviorTree | TEXT | 行为树 |
| OperationName | TEXT | 操作名称 |
| LuaScript | TEXT | Lua脚本 |
| UniquenessTag | TEXT | 唯一标识符 |
| WinnowFunction | TEXT | 剪枝函数 |

### BoostNames
- 初步研究猜测：
    - BoostNames数据表定义了所有可能出现在Boosts数据表中的触发增益类型。
    - 每个触发增益类型都有一个唯一的名称，并且与之相关联的值通常为1或0。
    - 具体而言，BoostNames数据表也可以为BoostHandlers和其他类似的表提供引用信息，从而保证数据表间的关联性

| 列名 | 数据类型 | 描述 |
| ---- | -------- | ---- |
| BoostType | TEXT | 触发增益类别名称 |
| BoostValue | INTEGER | 触发增益值 |
